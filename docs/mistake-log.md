# 错题池

> 这是项目内容的**主要燃料**(但不是唯一)。新关卡的触发有两条路径:
>
> - **(A) 错题驱动**:爸爸拍照 → Claude 转写到这里(追加,不重写)→ 打标签 → 累积到 ≥ 3 条同标签 → 提议做新关卡
> - **(B) 主题点名**:爸爸在对话中直接描述某类题目较弱 → 直接讨论 → 生成 → 事后在 [tags.md](tags.md) 记录
>
> 这个文件只管 (A)。两种路径详见 [../CLAUDE.md](../CLAUDE.md) "内容来源" 章节。

## 条目格式(给 Claude 看)

每条用如下结构。**追加在文件末尾**,不要插在中间(保持时间顺序)。

```markdown
### #NNN · YYYY-MM-DD · 来源:[作业 / 单元测验 / 课堂练习 / 口算 / 妈妈出题 / 其他]

- **题目**:(原题文字。如果图里有图形/数轴等,用文字尽量描述清楚)
- **她的做法**:(她写下/口述的过程。看不出来就写"直接写了答案 X")
- **正确答案**:(标准答案)
- **错在哪**:(一句话指出错点。例如:"把'追上'理解成'相遇'")
- **标签**:`tag-1`, `tag-2` (从 [tags.md](tags.md) 选;没合适的就新建,同步加到 tags.md)
- **根因猜测**:(可选但推荐。如果错点的根源是另一个更基础的概念,写出来。例:"看似是分数加减错,根源可能是约分不熟")
- **难度感受**:简单 / 中等 / 难 (从她的角度)
- **优先级**:🔴高 / 🟡中 / 🟢低 (高=她明显困惑,且这个概念基础重要)
- **后续**:_留空,做完关卡后回填(例:"2026-05-20 生成 fractions 岛 Lv2 后,她在变式题做对了")_
```

---

## 转写规范(给 Claude 看)

1. **追加,不重写**——每次只 append 新条目,不要重排或修改旧条目
2. **编号连续**——`#NNN` 取上一条 +1
3. **日期用今天**(`currentDate`,会在 system context 里告知)
4. **不会的就先空着**,不要瞎猜原始题目
5. 转写后,如果对应标签 ≥ 3 条,在回复里**主动提议**做新关卡
6. 如果根因猜测涉及一个更基础的概念,且这个根因标签也 ≥ 3 条,优先做根因关卡而不是表面错题关卡
